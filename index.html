API Documentation¶

GET /switcher/get_state¶

Get a device state

Returns the current state of a device

Query Parameters

  • id (string) -- the id of the selected device (Required)

  • ip (string) -- the ip address of the selected device (Required)

Example request:

GET /switcher/get_state?id=ab1c2d&ip=10.0.0.1 HTTP/1.1
Host: example.com
Status Codes

  • 200 OK --

    A JSON object representing the current state of the selected device

    Device ON:

    HTTP/1.1 200 OK
Content-Type: application/json

{
    "state": "ON",
    "time_left": "01:15:32",
    "time_on": "00:14:28",
    "auto_shutdown": "02:30:00",
    "power_consumption": 1274,
    "electric_current": 16.4
}

    Device OFF:

    HTTP/1.1 200 OK
Content-Type: application/json

{
    "state": "OFF",
    "time_left": "00:00:00",
    "time_on": "00:00:00",
    "auto_shutdown": "02:30:00",
    "power_consumption": 0,
    "electric_current": 0.0
}

  • 500 Internal Server Error --

    A JSON object hinting for the error

    Example for error when failed to get the current device state:

    HTTP/1.1 500 Internal Server Error
Content-Type: application/json

{
    "error": "an error of some type was raised"
}
